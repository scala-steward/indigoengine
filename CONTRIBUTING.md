# Contributing guidelines

All efforts to contribute to Indigo are welcome and encouraged, as the project can only be better with more eyes looking for problems and more minds trying to solve them.

All contributions must be in-line with our code of conduct.

Ways to contribute:

## Reporting issues

Think you've found a problem? We'd like to hear about it!

How to report via the [issue tracker](https://github.com/PurpleKingdomGames/indigoengine/issues):

> Please raise ONE TICKET PER ISSUE.

1. Try and search for similar issues currently open.
2. If you can't find an issue, raise a new one.
3. Please try to describe a minimal scenario that reproduces the issue.

## Updating documentation

Game engines and web frameworks are complicated, especially for new people, and so are only as good as their documentation. If you like to help update our docs, please raise a Pull Request against the documentation website repos:

- Indigo: https://github.com/PurpleKingdomGames/indigo-docs
- Tyrian: https://github.com/PurpleKingdomGames/tyrian-docs
- Ultraviolet: https://github.com/PurpleKingdomGames/ultraviolet-docs
- Roguelike-Starterkit: https://github.com/PurpleKingdomGames/roguelike-starterkit-docs

## Contributing code

The main authors have a general idea for the direction of our projects. Having said that, we do want people to feel empowered to join in!

Our policy in general is to always try to accept help where it is offered, but you can make it easier for us to do that.

General guidelines:

1. Only fix one thing at a time.
2. Small, easy to digest, minimal change Pull Requests are the easiest for us to merge.
3. Discuss, discuss, discuss. The bigger the change, the more important that it is to tell us what you're up to.

### Minor improvements

Correcting a misleading comment? Spotted a couple of lines that look off? Adding tests to existing code without changing the code itself? Please go right ahead and raise a PR.

### Larger changes

Please raise an issue on the project, explain what you intend to do, and indicate that you'd like to work on the fix.

### Pull requests without issues

We understand that sometimes it's just as easy to do the Pull Request as it is to write the issue, and it removes the wait time for us to get back to you with a thumbs up.

However, please appreciate that while we'd like to accept your assistance, if we haven't given you the nod to proceed, we may reject the work.

### Drive-by PRs

For clarity, Drive-by PRs are here defined as unsolicited PRs that provide improvements that appear somewhat random.

We'd like to have contributions from engaged members of our little community, but we do get contributions out of the blue from time to time. Generally this will follow the same rules as 'Pull requests without issues'.

As long as the PR genuinely adds value, even if it is a little random, we'll try and say yes.

### AI usage

This project was started long before AI coding assistants appeared, and so is not vibe coded, however the authors of this project do now use AI tools to _assist_ in it's development. As such, it would be hypocritical of us to reject contributions that have had AI assistance, and frankly, spotting AI produced code can be difficult.

The tricky part is that we have all the time in the world for human contributors who are trying their best, happy to help and grateful for your time investment, but very little patience for AI generated code that doesn't hit the mark.

Here are some guidelines:

- Respect our time. Generating code might be quick for you, but we still have to review it manually.
- All code contributed must be attributable to a 'reasonable' looking user. What does reasonable looking mean? Must appear to be a person, not an assistant. Assessed on a case by case basis.
- If your submission looks generated, we may push back and question it harder than normal.

## Donations and sponsorship

Contributions are based on time or money, and building and maintaining Indigo requires both. If you'd like to help cover our costs, help pay for development or just buy us a cup of coffee, we'll be very grateful.

Please look for the sponsorship button on the [indigo github repository](https://github.com/PurpleKingdomGames/indigo).
